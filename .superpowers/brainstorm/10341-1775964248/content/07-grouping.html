콘텐츠 그루핑 — 50만 노드를 어떻게 묶을까?

세계관(~100개)은 타입별로 충분하지만, 캔버스 노드(10K~500K)는 계층이 필수

세계관 — 이미 구조가 있음

세계관
  ├── 세계 (12)
  ├── 캐릭터 (24)
  ├── 장소 (8)
  ├── 세력 (4)
  └── 코덱스 (6)
→ 타입별 그루핑으로 충분. 최대 수백 개.

캔버스 — 여기가 문제

현재 캔버스 노드에 어떤 계층 구조가 있나요?

A

캔버스 단위 (Canvas → Nodes)

프로젝트에 여러 캔버스가 있고, 각 캔버스 안에 노드들이 있다면:
프로젝트 → 캔버스(챕터1, 챕터2...) → 노드들
캔버스가 자연스러운 그루핑 단위. 챕터별로 번역 진행률 관리 가능.

B

노드 타입 단위 (대화 / 나레이션 / 분기 / 조건)

캔버스 구분 없이, 노드 타입으로 그루핑:
대화 노드 (30만) → 나레이션 (10만) → 분기 (5만) → ...
같은 유형끼리 모아서 일괄 번역. 맥락은 약하지만 효율적.

C

캔버스 + 노드 타입 (2단 계층)

캔버스 → 노드 타입 → 노드 목록:
챕터1 캔버스 → 대화(500) / 나레이션(200) / 분기(50)
맥락(챕터) + 효율(타입별 일괄) 둘 다. 가장 세밀한 그루핑.

D

다른 구조가 있어요

캔버스 노드의 실제 구조를 알려주세요. 현재 데이터 모델에서 노드가 어떤 계층에 속하는지에 따라 달라집니다.
