도구 네비게이션 에디터 핵심 사용자
Crowdin 프로젝트 → 파일 → 문자열 → 언어 좌 원문 / 우 번역 (2열) 번역가 팀
Lokalise 프로젝트 → 키(문자열) → 언어 탭 키 선택 → 모든 언어 한 번에 개발팀+번역가
POEditor 프로젝트 → 언어 → 용어 목록 인라인 편집 (테이블 내) 1인/소규모
Unity L10n 테이블 → 키 → 언어 컬럼 스프레드시트 스타일 게임 개발자
articy:draft 엔티티(캐릭터/장소) → 속성별 번역 속성 패널 내 인라인 시나리오 작가

핵심 인사이트

💡 1인 작가에겐 "언어 중심"이 더 자연스럽다
POEditor가 소규모에서 인기 있는 이유 — "영어 먼저 끝내자"가 작가의 사고방식.
Crowdin의 파일 중심은 번역가 팀용.
💡 하지만 Flotter는 "세계관"이 있다
articy:draft처럼 엔티티(캐릭터/장소) 구조가 이미 있다.
"아르테미아의 영어 번역"이 자연스러운 진입점.
💡 AI 번역은 "일괄 → 검수" 패턴
Crowdin/Lokalise 모두 AI 초벌 → 사람 검수 워크플로우.
"번역 시작" 한 번 → 전체 AI 초벌 → 작가가 검수.
💡 에디터는 2열이 표준
좌측 원문 / 우측 번역이 업계 표준.
맥락 패널(사전, 이전 번역)은 우측 또는 하단.

Flotter에 맞는 구조 — 3가지 안

리서치 기반으로 재정리한 옵션입니다

A

언어 진입 + 콘텐츠 필터 (POEditor식)

대시보드(언어 카드) → 언어 클릭 → 세계관/캔버스 필터로 전환
작가: "영어 번역 들어가자" → 세계관/캔버스 탭으로 전환 → AI 일괄 번역 → 검수. 가장 단순한 흐름.

B

콘텐츠 트리 + 언어 컬럼 (articy식)

세계관 트리(세계/캐릭터/장소) → 엔티티 선택 → 언어별 번역 컬럼
작가: "아르테미아 캐릭터의 번역 상태" → EN/JA/ZH 한눈에. 세계관 일관성에 유리. 캔버스는 별도 섹션.

C

하이브리드 (대시보드 → 언어 → 콘텐츠 트리)

대시보드(전체 현황) → 언어 선택 → 콘텐츠 트리(세계관/캔버스) → 에디터
진입은 언어 중심(작가 사고방식), 작업은 콘텐츠 중심(세계관 구조). 두 장점을 결합. 3단계 깊이.
