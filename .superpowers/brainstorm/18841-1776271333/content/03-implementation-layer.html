구현 레이어 — 어디에 만들 것인가?

엔진(WebGL Canvas) vs 앱(React DOM) vs 하이브리드

A. 전부 엔진 (Canvas/WebGL)

메뉴, 태그, 아바타 모두 Canvas2D/WebGL로 렌더링. 엔진 template 시스템 확장.

Pros

  • 100만 노드에서도 일관된 성능
  • 줌/팬 시 메뉴가 자연스럽게 따라감
  • 엔진 독립성 유지 (React 없이 동작)

Cons

  • 메뉴/드롭다운을 Canvas로 만드는 건 복잡
  • 접근성(a11y) 직접 구현 필요
  • 텍스트 입력 = WebGL 텍스트 에디터 필요

B. 하이브리드 (추천)

노드 카드 = 엔진(Canvas)에서 렌더. 메뉴/팝업 = React DOM 오버레이로 띄움.

Pros

  • 노드 렌더링 성능 유지 (엔진)
  • 메뉴는 DOM이라 구현 쉽고 접근성 좋음
  • shadcn 컴포넌트 재활용 가능
  • tldraw, Figma가 쓰는 검증된 패턴

Cons

  • Canvas↔DOM 좌표 변환 필요
  • 줌 레벨에 따른 위치 계산

C. 전부 React DOM

노드 자체를 React 컴포넌트로 렌더. 메뉴도 DOM.

Pros

  • 구현 가장 빠름
  • shadcn 풀 활용

Cons

  • 100만 노드 60fps 불가능
  • 엔진 존재 이유가 사라짐
  • React Flow랑 차이 없음
