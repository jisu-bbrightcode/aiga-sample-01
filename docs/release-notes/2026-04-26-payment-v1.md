# Payment System v1 — Release Notes (2026-04-26)

## Goal
Polar.sh 단일 PSP 로 글로벌 USD 구독 + 사용량 크레딧 결제 시스템 출시.

## Highlights
- 11 Drizzle 테이블 (plans / subscriptions / credit_ledger / coupons / audit_log + 6)
- 8 NestJS services (PolarAdapter / CreditLedger / Subscription / Dunning / Coupon / Notification / Audit / WebhookRetry)
- 42 tRPC procedures (public 3 + auth 10 + admin 29)
- Webhook 처리 (Fastify raw + HMAC standardwebhooks + 13 Polar event types)
- 3 cron jobs (dunning daily / data-purge daily / webhook-retry 5min)
- Admin UI 11 routes (apps/admin)
- 사용자앱 UI 9 routes (apps/app)
- 6 react-email transactional templates
- ~125 service tests + e2e specs (Phase 14 + Phase 15 wiring)

## Catalog (sandbox)
- Plans: Free / Pro $19.99/mo / Pro $199.99/yr / Team $29.99/mo / Team $299.99/yr (5 row)
- Top-up SKU: 1k cr/$10, 5k cr/$40, 20k cr/$120

## Migration
- New tables (additive only): payment_* (11 tables)
- Drizzle migration: 0007_abandoned_mystique.sql + 0008_black_mandrill.sql
- Catalog seed: `pnpm --filter @repo/drizzle db:seed:payment`
- ENV new: POLAR_ACCESS_TOKEN / POLAR_ENV / POLAR_ORGANIZATION_ID / POLAR_WEBHOOK_SECRET

## Known limitations (deferred to v1.1+)
- discount.applied event handling (Polar emits via checkout.updated/order.paid metadata)
- changePlan downgrade deferred-intent table (currently no-op for downgrade direction)
- Polar drift reconcile cron (TODO marker in reconcile.cron.ts)
- audit_log hash chain
- bounce/spam tracking via Resend webhook
- 한국 PG 추가
- license/affiliate/referral domains
- mrrDelta30d 정확 계산 (snapshot table 필요)

## Production checklist before launch
1. Polar Production org 생성
2. Production token + webhook secret 발급, 환경변수 적용
3. Resend SPF/DKIM/DMARC 도메인 인증
4. `POLAR_WEBHOOK_SECRET` 채워서 controller enable
5. `pnpm --filter @repo/drizzle db:push` (production DB)
6. `pnpm --filter @repo/drizzle db:seed:payment` (Polar product id production 으로 새로 등록 후)
7. 사용자앱 `/pricing` 비로그인 접근 verify
8. Admin `/payment` 접근 verify (owner/admin role 필요)
