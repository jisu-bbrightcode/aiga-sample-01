# 비즈니스 스펙

이 디렉토리에 비즈니스 스펙 문서를 배치하면 에이전트가 인식합니다.

## 사용법

1. 외부에서 작성한 스펙 문서를 이 디렉토리에 배치
2. 에이전트에게 스펙 재정렬 및 구현 계획 작성 요청
3. 구현 계획 리뷰 후 승인하면 에이전트가 구현 시작

## 구조

```
docs/specs/
├── README.md                          ← 이 파일
├── booking-system.md                  ← 비즈니스 스펙 예시
└── booking-system/
    └── implementation-plan.md         ← 재정렬된 구현 계획
```

## 규칙

- 스펙 수용 절차: `.claude/rules/spec-intake.md` 참조
- 에이전트는 스펙을 프로젝트 룰에 맞게 재정렬한 후 구현합니다
