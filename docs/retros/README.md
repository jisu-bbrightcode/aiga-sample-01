# Team Retrospectives

> 팀별 회고 공간. 세션/스프린트/마일스톤 완료 시 작성.

## 구조

```
docs/retros/
├── README.md           ← 이 파일
├── engine/             ← Engine Team (Cyrus)
├── platform/           ← Platform Team (Felix)
├── growth/             ← Growth Team (Luna)
└── templates/
    └── retro-template.md
```

## 회고 작성 규칙

1. **파일명**: `YYYY-MM-DD-{주제}.md`
2. **작성 시점**: 마일스톤 완료, 큰 버그 수정 세션 후, 스프린트 종료
3. **필수 섹션**: 무엇을 했나 / 무엇이 잘 됐나 / 무엇이 잘못 됐나 / 배운 것 / 다음에 바꿀 것
4. **솔직하게**: 실패를 숨기지 않는다. 실패에서 배운 것이 가장 가치 있다.
