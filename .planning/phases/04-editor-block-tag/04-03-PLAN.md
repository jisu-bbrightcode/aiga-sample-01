    **Step 2: Rewrite use-inline-document-editor.tsx**

    Replace the entire 360-line file with a slim version (~60-80 lines, significantly reduced from 200+ of boilerplate logic). The hook should:

    1. Keep the same exported interface (`UseInlineDocumentEditorOptions`, `UseInlineDocumentEditorResult`) — no consumer API change.

    2. Keep `DOCUMENT_INLINE_EDITOR_ENABLED` flag and `DOCUMENT_EDITOR_EXTENSIONS` array at module level (these are consumer config, not boilerplate).

    3. In the hook body:
       - Create `EditorOverlayManager` in a ref (lazy init on first engine)
       - `handleInit`: register `editorHandler` via `engine.blockTagRegistry.register(editorHandler)`, call `setEditorConfig(DOCUMENT_TYPOGRAPHY, DOCUMENT_THEME_COLORS)`, create EditorOverlayManager with config from document constants
       - `openInlineEditor`: call `manager.open(nodeId, doc, scrollTop)`
       - The overlay JSX element is no longer needed — EditorOverlayManager creates its own DOM
       - `overlay` in the return value should be `null` (manager handles DOM internally)
       - `renderedNodes`: keep the _scrollTop injection logic for the atlas readonly paint (the editor handler reads this)
       - Wire `onCommit` to apply `applyUpdateDocumentData` and call `onChange`
       - Wire `onScroll` to update `previewScrollRef`
       - `canEditNode`: same logic (check node type === "document" and onChange exists)
       - Cleanup: destroy manager on unmount

    4. Remove `resolveDocumentEditorScreenBounds` helper function entirely (absorbed into EditorOverlayManager._resolveScreenBounds).

    5. Remove all direct DOM refs (`layerRef`, `canvasRef`), RAF management, syncBounds function, controller event wiring, document pointerdown listener — all absorbed into EditorOverlayManager.

    **What stays in the hook:**
    - Node type check (`canEditNode`)
    - Extension config (`DOCUMENT_EDITOR_EXTENSIONS`)
    - Story mutation (`applyUpdateDocumentData` on commit)
    - Scroll tracking for preview (`previewScrollRef` + `_scrollTop` injection in `renderedNodes`)
    - Manager lifecycle (create on init, destroy on unmount)
    - Double-click -> openInlineEditor routing

    **What is removed (absorbed into engine):**
    - `sessionRef`, `InlineDocumentSession` type
    - `closeRef` with commit/cancel logic
    - `layerRef`, `canvasRef` DOM refs
    - `syncBounds` RAF loop (all 40 lines)
    - `resolveDocumentEditorScreenBounds` helper (all 25 lines)
    - Controller creation with typography/theme config (15 lines)
    - Controller event wiring (change, scroll, blur) (15 lines)
    - Document-level pointerdown capture listener (10 lines)
    - Overlay JSX element (20 lines)
    - useEffect for active editor lifecycle (80 lines)
