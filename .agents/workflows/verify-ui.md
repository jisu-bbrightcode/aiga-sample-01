# Verify UI

브라우저 서브에이전트를 사용한 빠른 UI 검증 워크플로우입니다.

## Step 1: 대상 확인

사용자에게 어떤 페이지/컴포넌트를 검증할지 확인합니다.
URL이 명시되지 않았으면 최근 수정된 라우트 파일에서 경로를 추출합니다.

## Step 2: 브라우저 접속

해당 URL로 브라우저를 네비게이션합니다.
- User 앱: http://localhost:3000/{path}
- Admin 앱: http://localhost:3001/{path}

인증이 필요하면 먼저 로그인합니다:
- 로그인 페이지 접속 → qa@test.com / q1w2e3r4t5 입력 → 로그인

## Step 3: 시각적 확인

1. 스크린샷을 캡처하여 전체 레이아웃 확인
2. FeatureHeader (제목, 액션) 표시 여부
3. FeatureContents (메인 콘텐츠) 표시 여부
4. 데이터 목록/카드/테이블 등 핵심 요소

## Step 4: 콘솔 확인

브라우저 콘솔 메시지를 읽어 critical 에러가 없는지 확인합니다.
- Error 레벨: 조사 필요 → 이슈 보고
- Warning 레벨: 허용 (메모만)

## Step 5: 핵심 인터랙션

사용자가 가장 많이 사용할 인터랙션 1-3개를 테스트:
- 버튼 클릭 → 기대 동작 확인
- 폼 입력 → 제출 → 피드백 확인
- 네비게이션 → 올바른 페이지 이동

## Step 6: 결과 보고

```
### UI 검증 결과
- **URL**: {tested URL}
- **렌더링**: PASS / FAIL
- **콘솔 에러**: 없음 / {에러 내용}
- **인터랙션**: PASS / FAIL
- **스크린샷**: [캡처 첨부]
- **이슈**: {발견된 문제점 or 없음}
```
